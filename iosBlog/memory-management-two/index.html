Memory Management - Entry Two

This function is exactly the same as Ex. 1 except:
We have marked .customer on the Account object as weak.
When an object is marked as weak, it's reference count will not increment.
This will allow us to break our retain cycle and call the deinit functions for both objects when our customer and account objects are outside the scope of the do loop.
