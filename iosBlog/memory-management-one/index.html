Memory Management - Entry One

Apple's system for automatic tracking and managing of memory usage is called ARC (Automatic Reference Counting).
ARC is of great convenience to Apple developers because it automatically frees up memory when objects aren't referenced anymore (the object's reference count is zero). It used to be that iOS developers would have to keep track of reference count manually and also deallocate objects manually when the object's reference count reached zero. Luckily, ARC fixed that!
ARC only applies to reference types (typically classes), and never applies to value types (structs, enums, etc.).

One primary concern is the creation of retain cycles, which occur when two objects reference each other. If both objects reference each other, the reference count will always be at least one. Since any reference count higher than one will ensure an object will remain in memory, the object cannot be deallocated, and will live in memory even if the app stopped needing it a long time ago!
